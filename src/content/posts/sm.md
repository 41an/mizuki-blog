### 5.1. PCI 密码卡技术规范 
本规范的目标是为信息安全基础设施 PCI 密码卡设备制定统一的功能要求、硬件要求、软件要求（含应用接口要求）、安全性要求（含密钥结构）、检测要求等有关内容，通过统一的 API 函数集调用 PCI 密码卡设备，向上层提供基础密码服务，实现不同生产厂商的 PCI 密码卡设备互操作，方便应用。为该类密码设备的开发、使用及检测提供标准依据和指导，有利于提高该类密码设备的产品化、标准化和系列化水平。

- SM 3 是中国国密标准，等价于 SHA-256 安全等级。


在《GB/T 22239—2019 信息安全技术网络安全等级保护基本要求》《GB/T 20518—2006 信息安全技术密码术语》等文件里，密码技术能提供的安全服务通常包括：
	•	机密性（Confidentiality）
	•	完整性（Integrity）
	•	认证（Authentication）
	•	不可否认性（Non-repudiation）

这四项就是标准定义的安全服务（Security Services）。也是密码学的基本属性。
密码学的两大分支是 `密码编码学` 和 `密码分析学`。

#### SM 2
私钥长度
256 位
公钥长度
- 在实际传输中，公钥通常用 **未压缩格式** 以 0 x 04 开头，然后跟 x、y 坐标，即总长度 65 字节（**520 位** ，含 8 位标识）。
- 如果采用压缩格式，只传 x 坐标 + 1 位符号 → 33 字节（264 位）。

素性检测主要用于选定椭圆曲线参数，而 `SM2 曲线参数固定`，因此无需做素性检测。


- SM 2 应用的领域：数字签名、密钥交换、公钥加密

- ECC 具备的技术：椭圆曲线点加、椭圆曲线点乘、散列值计算、随机数生成、素性检测
- SM 2 具备的技术：椭圆曲线点加、椭圆曲线点乘、散列值计算、随机数生成


- SM 2 签名：杂凑值计算、椭圆曲线点乘、随机数生成
- SM 2 验签：杂凑值计算、椭圆曲线点乘


- SM 2 仅签名的算法进入了 ISO 国际标准。

- SM 2 密码算法的安全强度大致相当于 `3072`  比特长度的 RSA 算法。

#### SM3
> MD 结构

SM 3 的设计参考了 SHA-256，但做了本地优化和常量替换以满足国密安全要求
因此虽然框架相似，但压缩函数内部计算完全不同

#### SM9
> 2016 年成为国标。2017 年成为国际标准。	
> 使用256位的BN曲线。

随机数 → MSK → 点乘生成 MPK → 用户公钥派生

（主私钥）MSK<-  随机数 
点乘生成 MPK（主公钥） <- MSK
用户公钥 ← 用户 ID + MPK 派生  
用户私钥 ← KGC 用 MSK + 用户 ID 生成


- SM9公钥加密算法是密钥封装机制和消息封装机制的结合
- SM9公钥加密算法消息封装机制使用了KDF作为辅助函数。
- SM9数字签名算法的辅助函数包括密码杂凑函数和随机数发生器。
- SM9密钥封装机制和公钥加密算法都需要密钥派生函数作为辅助函数。
- SM9密码算法系统参数由KGC选择。
- SM9密钥交换协议需要使用密码杂凑函数、密钥派生函数、随机数发生器作为辅助函数。
- SM9数字签名算法、密钥交换协议、密钥封装机制、公钥加密算法都需要使用密码杂凑函数和随机数发生器作为辅助函数。
