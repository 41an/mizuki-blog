## 1. 小程序
> 写在前头，微信 4 的小程序 `wxapkg` 的文件是动态加载的，不是一次性加载完的，特殊小程序还能特殊加载。因此建议使用微信 3。
### 1.1. 微信 3 的安装
[WeChat32 v3.9.12](https://dldir1v6.qq.com/weixin/Windows/WeChatSetup_x86.exe) 仍有较多调试工具无法直接使用，因此建议使用 [3.9.10.27版本](https://github.com/tom-snow/wechat-windows-versions/releases/download/v3.9.10.27/WeChatSetup-3.9.10.27.exe)，但安装成功后会发现提示版本过低无法登陆。
![image.png](http://imgur.41an.com/piclist/2026-01/WeChatVersion2Low-6b57042494ae49a302d7704acb1eae50.png)
#### 1.1.1. 修改临时变量
在 `WeChat.Exe` 目录下，创建如下内容的 `start.bat`
```powershell
@echo off
set "__COMPAT_LAYER=~ ARM64WOWONAMD64"
start "" "WeChat.exe"
```
双击运行即可。
#### 1.1.2. 修改注册表
创建如下内容的 `regWX.reg`，需注意
1. 微信文件 `WeChat.exe` 的路径为绝对路径
2. 以管理员身份运行
```powershell
Windows Registry Editor Version 5.00
[HKEY_CURRENT_USER\SOFTWARE\Microsoft\Windows NT\CurrentVersion\AppCompatFlags\Layers]
"C:\\PATH\\TO\\WeChat.exe"="~ ARM64WOWONAMD64"
```
双击运行后，即可直接正常使用 `WeChat.exe` 。

## 2. Android
### 2.1. 流量转发
Android 9 后的证书安装及流量转发会有些限制，因此建议使用早期版本如 Android 7。
#### 2.1.1. 在模拟器应用的设置里设置代理服务器
![image.png](http://imgur.41an.com/piclist/2026-01/androidStdioProxySetting-944573ff1dcd7bfe2047a1372829062f.png)
#### 2.1.2. 在模拟器里的设置里设置代理服务器
- 若为 NAT 子网则设置代理服务器为网关
- 若为桥接则为物理机地址
通常在 `Settings` --> `Network` --> `WIFI` --> `Proxy/Edit`

![image.png](http://imgur.41an.com/piclist/2026-01/androidProxySetting-8df394015bfa1be141a0385a0427bfc3.png)

![image.png](http://imgur.41an.com/piclist/2026-01/20260111022856184-1a2542ed881e4ea9d033f3aad5027f9c.png)
### 2.2. Apktool 反编译
> 主要根据 `resources.arsc` 、`Manifest` 还原应用资源结构，并直接将 `classes.dex` 反编译为 `smali` 代码。[Apktool-Releases](https://github.com/iBotPeaches/Apktool/releases)
```shell
# 反编译
java -jar apktool_2.9.3.jar d -f ./test.apk -o ./unpack
# 重建
java -jar /apktool_2.9.3.jar b -f unpack/ -o ./test.v1.apk
# 签名
java -jar signapk.jar certificate.pem key.pk8 ./test.v1.apk ./test.v1.sign.apk
```
### 2.3. Dex 2 jar 反编译
apk 的本质就是一个 ZIP 容器，先直接将应用解压得到 `dex` 文件，再去推理转化成 `jar` 包。
```shell
unzip ./test.apk
dex2jar classes.dex
jd-gui
```
## 3. CS 端
在 `Burp` 中可通过 [Nope-Proxy](https://github.com/summitt/Nope-Proxy) 实现对 `TCP` 协议流量的转发。

在 `Burp` 中导入插件。
![image.png](http://imgur.41an.com/piclist/2026-01/BurpAddPlugin-475e17e1b6b2c7438ae1c66df7b43bc0.png)

在 `Proxy settings` 中监听所有端口
![image.png](http://imgur.41an.com/piclist/2026-01/BurpAllInterfaces-9e0d08b5e916f6dcd3ea7dcfb866c59a.png)
-  `Server Address`：填写监听服务器的地址
-  `Server Port` ：填写监听服务器的端口
- `Listen Port`：填写本插件监听的端口
> btw，不要忘了点击 `Enable` 启用和 `Add 80 & 443 to Burp` 使用 `Burp Intruder` 一并分析 `HTTP` 流量。

![image.png](http://imgur.41an.com/piclist/2026-01/NoPESettings-d367c2194c8068a8094aea134edbedcb.png)

`Proxifier` 将服务器端口更改为 `8081`
![image.png](http://imgur.41an.com/piclist/2026-01/20260111154242489-240831ec5158825e7472b7691f2b4a9c.png)

将 `Action` 设置为对应的处理行为。
![image.png](http://imgur.41an.com/piclist/2026-01/SetAction-ab70dfe5e1aac183b38900f61ba915dd.png)
保存策略后即可成功监听。
![image.png](http://imgur.41an.com/piclist/2026-01/TCPIntercept-6c117c3c3dadf3b0e362b7bd91ff45ed.png)

如果仅需要监听操作，还可以使用 `Wireshark`。

```shell
# 查询端口号
C:\Users\alan>tasklist | findstr /i {task}

# 查看程序IP外联
C:\Users\alan>netstat -ano | findstr {pid}
```
